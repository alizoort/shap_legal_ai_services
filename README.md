# SHAP Legal AI Services

This repository now includes a proof-of-concept **Employment / HR Compliance Risk Detection** workflow inside `services/legal_ai_service`.

The POC is intentionally small and demo-oriented:

- stateless API
- synthetic dataset
- TF-IDF + Logistic Regression baseline
- SHAP explanations tied to the real classifier
- optional OpenAI commentary layered on top of the classifier output

It is **not legal advice** and **not production compliance automation**.

## Architecture

- `services/gateway_service`
  Proxies `/legal-ai/*` to the downstream service.
- `services/legal_ai_service`
  Hosts dataset scripts, model artifacts, the FastAPI analysis API, and SHAP-based explanations.
- `shap_service_runtime`
  Shared runtime helpers for auth, config, DB, and observability.
- `shap_contracts`
  Shared contracts package used by the workspace.

## Dataset

The classifier is trained on a committed synthetic corpus with:

- `120` training examples
- `18` gold validation examples
- balanced `low / medium / high` labels
- categories covering monitoring, termination, hiring, workplace policy, discrimination, compensation, and employee data usage

Rebuild the CSVs from the checked-in blueprint:

```bash
python3 -m services.legal_ai_service.scripts.export_synthetic_dataset
```

Generated files:

- `services/legal_ai_service/data/training.csv`
- `services/legal_ai_service/data/gold.csv`

## Training

Train and export the runtime artifacts:

```bash
python3 -m services.legal_ai_service.scripts.train_compliance_model
```

Artifacts are written to `services/legal_ai_service/model_artifacts/`:

- `vectorizer.joblib`
- `classifier.joblib`
- `background_texts.json`
- `metrics.json`
- `global_feature_importance.json`

The training script records:

- 5-fold cross-validation metrics
- holdout gold-set accuracy and macro F1
- confusion matrices
- per-class precision/recall/F1
- global SHAP feature importance

## API

Main routes exposed by `legal_ai_service`:

- `GET /legal-ai/ping`
- `GET /legal-ai/model-summary`
- `POST /legal-ai/analyze`

Example request:

```json
{
  "text": "We reserve the right to monitor all employee communications at any time without notice.",
  "include_commentary": false
}
```

Example response fields:

- `risk_label`
- `class_probabilities`
- `needs_human_review`
- `top_features_supporting_prediction`
- `top_features_against_prediction`
- `plain_english_explanation`
- `openai_commentary`

`needs_human_review` is `true` for all `medium` and `high` predictions, and also for `low` predictions when the top class probability is below `0.60`.

## Optional OpenAI Commentary

OpenAI is **not** used for classification.

It is only used when all of the following are true:

- `LEGAL_AI_ENABLE_COMMENTARY=true`
- `LLM_PROVIDER=openai`
- `LLM_API_KEY` is configured
- the client sends `include_commentary=true`

When commentary is disabled or unavailable, the core ML result still succeeds and `openai_commentary` is `null`.

## Limitations

- The dataset is synthetic and relatively small.
- Metrics are suitable for a POC, not for production claims.
- The model is a bag-of-words baseline and does not understand legal nuance the way a domain expert does.
- SHAP explains the ML model behavior, not legal correctness.
- Optional commentary may still be imperfect because it is generated by an LLM.
